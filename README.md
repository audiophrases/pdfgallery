# PDF Gallery

A static PDF gallery hosted on GitHub Pages. Each top-level folder in this repo becomes a gallery available at:

```
https://audiophrases.github.io/pdfgallery/<folder>/
```

The site renders thumbnails of the first page of each PDF using PDF.js, and the admin can attach comments that all visitors can see.

## Add a new gallery

1. Create a folder at the repo root (e.g. `c/`).
2. Drop one or more `.pdf` files into it.
3. Commit and push.

A GitHub Action ([`.github/workflows/build-manifests.yml`](.github/workflows/build-manifests.yml)) regenerates [`gallery.json`](gallery.json) automatically and commits it. The site will pick up the new folder once that commit lands and Pages rebuilds (usually under a minute).

No per-folder `index.html` is needed; the SPA fallback ([`404.html`](404.html)) handles routing.

## Admin and comments

- Click **Admin login** in the header. Password is hardcoded in [`assets/app.js`](assets/app.js) (`ADMIN_PASSWORD`).
- After logging in, click **Set GitHub token**. Paste a fine-grained Personal Access Token with **Contents: Read and write** permission for `audiophrases/pdfgallery`. Create one at https://github.com/settings/personal-access-tokens. The token is stored in the admin's browser `localStorage`.
- Each thumbnail then shows a comment badge. Click it to expand the panel, then post or delete comments. Posts commit changes to [`comments.json`](comments.json) via the GitHub API.
- A thumbnail with one or more comments shows the badge to all visitors. Visitors without admin access can read but not edit.

## Why a manifest?

Visitor reads (folder list, PDF list, comments) are static fetches against GitHub Pages — **no GitHub API calls**, so no 60/hr rate limit. Only the admin's *write* path touches the API, and it's authenticated (5,000/hr).

## Files

- [`index.html`](index.html), [`404.html`](404.html) — SPA shell.
- [`assets/app.js`](assets/app.js) — routing, manifest/comments fetching, thumbnail rendering, admin UI.
- [`assets/style.css`](assets/style.css) — styles.
- [`gallery.json`](gallery.json) — generated by the build action; lists folders and their PDFs.
- [`comments.json`](comments.json) — comment store, edited by the app via the GitHub API.
- [`scripts/build-manifests.mjs`](scripts/build-manifests.mjs) — scans folders, writes `gallery.json`.
- [`.github/workflows/build-manifests.yml`](.github/workflows/build-manifests.yml) — runs the build script on push.

## Local dev

```sh
node scripts/build-manifests.mjs   # regenerate gallery.json from local folders
npx http-server . -p 8080          # serve at http://localhost:8080/
```
